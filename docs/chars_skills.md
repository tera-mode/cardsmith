# cardsmith — 60キャラ × スキル割り振り設計書

35スキル（既存5＋新規30）を60キャラに割り振った最終配備案。属性間の総コスト・スキル多様性・役割分布のバランスを設計。

## 設計バランスの原則

1. **総コストを揃える** — 各属性の10体合計コストは125〜138に収束（±10%以内）。属性をどう組んでも公平に戦える
2. **コスト分布を統一** — 各属性で 低×2 / 中×4 / 高×3 / 極×1 の4階層
3. **役割の多様性** — 各属性10体内に「アタッカー／タンク／支援／機動／特殊」がバラけて配置
4. **スキル親和性を尊重** — 親和★★★のスキルを各属性のキー戦略カードに集中配置
5. **属性カラーの強化** — 聖＝守る／冥＝死で削る／森＝粘る／焔＝壊す／蒼＝操る／鋼＝計算する を肌で感じる配備
6. **スキル被りはOK** — 同じスキルが別属性で使われるのはフレーバー差で吸収（聖の治癒=祝福、森=樹液、鋼=修理）

→ **属性ごとの「強み」が明確に分離**。じゃんけん的な相性関係が成立しうる：
- 焔（火力）→ 森（耐久）が苦手だが冥（低HP）に強い

- 冥（除去）→ 聖の支援系を沈黙で潰せる
- 鋼（バランス）→ 突出した弱点なし、突出した強みもなし

## 実装メモ

### cards.ts への適用

新規追加が必要なフィールドはなし。既存の `Card` 型でそのまま記述可能：

```typescript
{
  id: 'sei_noa',
  name: '見習い聖騎士ノア',
  attribute: 'sei',  // 新規フィールド追加（属性管理用）
  cost: 4,
  movement: { type: 'step', directions: [{ dx: 0, dy: -1 }] },
  attackRange: { type: 'step', directions: [{ dx: 0, dy: -1 }] },
  atk: 1,
  hp: 1,
  // skill省略
}
```

### スキル実装の優先順位（既存5+追加5でMVP）

Phase 1で実装する10スキル：既存5＋業火・凍結・残骸・吸血・自爆

これだけで以下のキャラが動く：
- 聖：ノア・エルナ(慧眼ﾅｼ→なし)・リーゼ・グレイル(殉教ﾅｼ→なし) ＋ 治癒(ミレイユ)
- 冥：カル・ヴェラ(吸血)・ノクト(凍結)
- 森：ヒナ・リル・カイ(残骸) ＋ 治癒(アルカ)
- 焔：ココ・ロン・ヴァル(業火) ＋ 貫通(ブラン)・反撃(バハムート)
- 蒼：カイト・ナージャ ＋ 凍結(ライカ)
- 鋼：MK-01 ＋ 自爆(ゼロ)・大貫通(タイタン)

→ Phase 1で **31キャラ** が動作可能（60体中の半分超）。残り29キャラはPhase 2以降のスキル追加に連動して順次有効化。
